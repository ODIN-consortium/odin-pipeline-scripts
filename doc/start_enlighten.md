# start_enlighten.sh

`start_enlighten.sh` is a Bash script that starts (or restarts) the Enlighten web visualization platform using Docker Compose. Enlighten provides an interactive web interface for exploring pipeline results (taxonomic profiles, AMR data, Biomeme qPCR data).

## Purpose

- Starts the Enlighten API and client containers.
- Mounts pipeline output data (Feather files) into the containers for visualization.
- Makes the web interface available at http://localhost.

## Usage

```bash
start_enlighten.sh [enlighten_data_path]
```

- `enlighten_data_path`: Absolute path to the directory containing Feather data files. Defaults to `enlighten/data` if not provided.

Use `-h` or `--help` for usage instructions.

## How It Works

1. Resolves the Enlighten data path from: the command-line argument, the user config (`~/.odin_ml/odin_paths.txt`), or the default (`enlighten/data`).
2. Exports `HOST_DATA_PATH` and `HOST_PUBLIC_FILES_PATH` as environment variables for `docker-compose.yml`.
3. Verifies the data directory exists. Creates the public files directory if needed.
4. Runs `docker compose down` followed by `docker compose up -d` in the `enlighten/` directory.

## Docker Compose Services

The `enlighten/docker-compose.yml` defines two services:

| Service | Image | Port | Purpose |
|---------|-------|------|---------|
| `enlweb-api` | `norceresearch1/enlweb-api:odin2.0` | 8081 → 8080 | REST API serving data |
| `enlweb-client` | `norceresearch1/enlweb-client:odin2.0` | 80 → 80 | Web frontend |

## Output

- Enlighten visualization available at **http://localhost**
- API available at **http://localhost:8081**

## Requirements

- Docker and Docker Compose installed
- Feather data files in the data directory (generated by `create_kraken_datasets.py`, `parse_amr_json.py`, or `traverse_biomeme.py`)

## Example

```bash
# Start with default data path
start_enlighten.sh

# Start with custom data path
start_enlighten.sh /mnt/c/ODIN/data/enlighten
```
