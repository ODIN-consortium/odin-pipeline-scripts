ODIN Pathogen Explorer is an integrated platform for the analysis, visualization, and interpretation of pathogen detection data from environmental and clinical samples.

Datasets generated by the ODIN pipelines, including Taxprofiler for sequencing data and Biomeme for qPCR data, are processed and standardized for use in the ODIN Pathogen Explorer.

Overview

ODIN provides tools for:

  • Processing raw sequencing and qPCR data
  • Generating standardized datasets for downstream analysis
  • Visualizing results interactively
  • Integrating metadata for contextual interpretation

The platform is designed to be modular and extensible, supporting a range of pathogen detection assays and analysis pipelines.

Taxprofiler Datasets

Taxprofiler is an nf-core pipeline for taxonomic profiling of sequencing data, particularly from Oxford Nanopore platforms. ODIN integrates Taxprofiler outputs to provide:

  • Species-level abundance tables
  • Pathogen detection summaries
  • Interactive visualizations of taxonomic composition

Processed Taxprofiler datasets are standardized and merged with sample metadata, enabling comparative analysis across runs, locations, and timepoints.

Taxprofiler Dataset Contents

A typical Taxprofiler dataset includes the following columns:

  • Sample Name: Identifier for the sequencing sample
  • Barcode: Nanopore barcode or sample label
  • Taxon: Detected taxonomic entity (species, genus, etc.)
  • Taxon Rank: Taxonomic rank (e.g., species, genus)
  • Taxon ID: NCBI taxonomic identifier
  • Reads: Number of reads assigned to the taxon
  • Relative Abundance: Proportion of reads assigned to the taxon
  • Pathogen Flag: Indicates if the taxon is a known pathogen (based on reference database)
  • Metadata Fields: Additional contextual information (e.g., location, date, run name)

These datasets enable users to explore taxonomic profiles, identify pathogens, and compare results across samples and runs.

Biomeme Datasets

Biomeme qPCR assays are used for targeted detection of pathogens in environmental DNA (eDNA) samples. ODIN processes Biomeme qPCR results using the decode.py script, which:

  • Extracts Cq values and assay information from Excel files
  • Calculates abundance estimates using assay parameters
  • Assigns quality flags to each measurement for data reliability
  • Merges results with sample metadata

Biomeme datasets are integrated into ODIN for visualization and filtering, allowing users to assess detection confidence and compare results across assays and samples.

Biomeme Dataset Contents

A typical Biomeme dataset includes the following columns:

  • Date: Assay date
  • Run Name: Name of the qPCR run
  • Well Number: Well identifier
  • Biomeme replicate: Replicate/sample name
  • Fluorophore: Detection channel
  • Target ID: Assay target identifier
  • Cq: Quantification cycle value
  • Abundance: Calculated abundance based on assay parameters
  • Abundance diluted: Abundance adjusted for dilution factor
  • Mean Abundance: Mean abundance for replicate/target
  • quality_flag: Data quality indicator (see below)

To read more about using enlighten for data analysis view the BiomemeAssay Explorer document in the BIOMEME tab.

Quality Flag Description

Flag Value Meaning
0 Not checked (default, before flagging logic is applied)
1 Good: More than one non-zero Cq value for the replicate/target combination
2 Questionable: Only one non-zero Cq value for the replicate/target combination
3 Bad: Cq value is zero (no amplification detected)
4 Below NTC threshold: Cq value is below the NTC (No Template Control) threshold for matching fluorophore and target ID

The flag enables filtering and interpretation of qPCR results based on data quality, helping users focus on reliable detections.

Using Templates in ODIN Pathogen Explorer

ODIN Pathogen Explorer supports the use of templates to customize the display and options for datasets based on their names. Templates are defined in the ServerConfig.json file using regular expressions to match dataset names.

To use the template feature for datasets in the front end:

  1. Locate Your Dataset
    In the table view, find the dataset whose name matches the template regex configured in ServerConfig.json.
  2. Find the Template Icon
    Next to the matching dataset, look for the template icon:
  3. Click the Icon
    Click the open_in_new icon to apply or view the template associated with that dataset.
  4. View Customized Display
    The dataset will update its display or options according to the template settings.

Notes:

  • The open_in_new icon only appears for datasets matching the configured regex.
  • Templates control formatting, columns, and other visualization options.
  • If you do not see the icon, check your server configuration and refresh the page.
